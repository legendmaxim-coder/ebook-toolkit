# eBook Toolkit — PWA / Capacitor scaffold

Минимальный прототип для Android (гибрид): PWA фронтенд + опциональная упаковка через Capacitor.

## Функции прототипа
- **Поддерживаемые форматы:** `.fb2`, `.epub`, `.html`, `.htm`, `.txt` и архивы `.zip` (внутри — `.fb2`, `.epub` или `.mobi`).
  - MOBI требует конверсии через GitHub Actions (см. ниже).
- **Сортировка файлов** (кнопки ↑/↓) перед объединением.
- **Объединение нескольких книг в один EPUB** с иерархией глав:
  - **Сохранение структуры:** разделы/главы каждой исходной книги остаются отдельно в навигации (TOC/NCX).
  - **Содержание:** автоматически генерируется список всех книг и глав.
  - **Метаданные и обложка:** обложка берётся из первой загруженной книги (если есть).
- **Экспорт в EPUB:** единственный формат для скачивания (поддерживается Kindle, любыми читалками).

## Поддержка MOBI, AZW3 и других форматов
- **MOBI, AZW3, PDF и другие:** эти форматы требуют нативной конверсии (невозможно парсить в браузере без сложных WASM портов).
- Используйте **GitHub Actions workflow** `.github/workflows/convert.yml` (включен в проект) для конверсии:
  - Положите файлы (`.mobi`, `.pdf`, `.azw3` и др.) в папку `input/` в репозитории.
  - Откройте Actions → выберите workflow `Convert ebooks to EPUB` → нажмите «Run workflow».
  - По завершении скачайте артефакт `converted-epubs` с готовыми `.epub` файлами.
  - Загрузите полученные EPUB в основное приложение для объединения с другими книгами.

**Примечание:** GitHub Actions бесплатны для публичных репозиториев. Приватные репозитории получают ограниченное количество бесплатных минут, но обычно достаточно для конверсии нескольких книг в месяц.

## Типовой поток использования

### 1. Локальное объединение FB2/EPUB
- Запустите приложение локально: `npx http-server -c-1 . -p 8080`
- Откройте http://localhost:8080 в браузере.
- Загрузите файлы (`.fb2`, `.fb2.zip`, `.epub`).
- При необходимости переставьте порядок кнопками ↑/↓ (кнопки в начале каждой строки).
- Нажмите **«Объединить выбранные»**, затем **«Скачать EPUB»**.
- Откройте скачанный EPUB в Kindle, читалке на телефоне/ПК — видно будет:
  - Обложка (если была в первой книге)
  - Содержание со всеми книгами и их главами
  - Каждая глава как отдельная страница в навигации

### 2. Конверсия MOBI/PDF в EPUB через GitHub Actions
- Создайте публичный GitHub репозиторий (или используйте приватный).
- Положите `.mobi`, `.pdf`, `.azw3` файлы в папку `input/` и сделайте push.
- Откройте Actions → workflow `Convert ebooks to EPUB` → «Run workflow».
- По завершении (обычно 1-2 минуты) скачайте артефакт `converted-epubs`.
- Загрузите полученные `.epub` в основное приложение для дальнейшего объединения.

## Быстрый старт (локально)
1. Установите `http-server` и запустите:
```bash
npx http-server -c-1 . -p 8080
```
2. Откройте http://localhost:8080 в браузере.

## Capacitor (обёртка для Android)
1. Установите зависимости и Capacitor в проекте Node (в каталоге с `package.json`):

```bash
npm init -y
npm install @capacitor/core @capacitor/cli --save
npx cap init ebook-mobile com.example.ebook
```

2. Соберите веб-активы (в данном прототипе — просто файлы в корне). Затем добавьте Android и синхронизируйте:

```bash
npx cap add android
npx cap copy
npx cap open android
```

3. В Android Studio соберите APK и установите на устройство.

## Ограничения и дальнейшие шаги
- EPUB генерируется с базовой структурой TOC/NCX; можно расширить для лучших читалок (добавить cover image в spine и т.д.).
- Для конверсии в MOBI/AZW3 нужен Calibre (нельзя выполнить в браузере). Используйте GitHub Actions (см. выше).
- При объединении больших файлов могут быть ограничения памяти браузера; для улучшения рекомендуется добавить стриминг обработки.
